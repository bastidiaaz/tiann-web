## Deviations from Brief

### 1. TypeScript upgraded from 4.x → 6.0.3 (required, not optional)
The brief's `tsconfig.json` uses `"moduleResolution": "bundler"` and `"allowImportingTsExtensions": true`, both of which were introduced in TypeScript 5.0. The project had TypeScript 4.6.3 installed. Running `npx tsc --noEmit` with the brief's exact tsconfig produced:

```
tsconfig.json(8,25): error TS6046: Argument for '--moduleResolution' option must be: 'node', 'classic', 'node12', 'nodenext'.
tsconfig.json(9,5): error TS5023: Unknown compiler option 'allowImportingTsExtensions'.
```

TypeScript was upgraded to `^6.0.3` (latest at time of task) to satisfy the brief's tsconfig requirements. This is a necessary deviation — the brief's exact tsconfig cannot work on TS 4.x.

### 2. `src/vite-env.d.ts` created (not in brief, but required)
After deleting `src/react-app-env.d.ts` (which provided CSS module types for CRA), TypeScript produced:

```
src/index.tsx(3,8): error TS2882: Cannot find module or type declarations for side-effect import of './index.css'.
```

The standard Vite fix is to add `/// <reference types="vite/client" />` in `src/vite-env.d.ts`. This is the idiomatic equivalent of CRA's `react-app-env.d.ts` and is created by Vite's own scaffolding tool. It was not mentioned in the brief but is required for a clean TypeScript build.

### 3. `--legacy-peer-deps` was needed for npm operations
The brief predicted that removing `react-scripts` would resolve peer dep conflicts. In practice, the conflicts (from `@react-spring/*` packages requiring React 16/17 while the project uses React 18) persist after removal. All npm commands required `--legacy-peer-deps`.

### 4. `package-lock.json` committed
A `package-lock.json` was generated by npm operations and committed. The old `yarn.lock` was deleted as specified.

## Concerns

**TypeScript 6.0.3** — This is a major jump from 4.x. TypeScript 6 is newly released and some third-party type definitions (particularly `@types/react@^17`, `@types/react-dom@^17`, `@types/three@^0.139`) may lag behind. `skipLibCheck: true` in the tsconfig mitigates this, and the `tsc --noEmit` check passed cleanly, but downstream tasks (Task 2+) should be aware that TypeScript is now 6.x rather than 4.x.

**Persistent peer dep conflicts** — The `@react-spring` packages (transitive from `@react-three/drei`) have peer requirements for React 16/17, while the project uses React 18. This conflict will remain until those deps are upgraded. Task 2 (likely upgrading Three.js ecosystem deps) would be the appropriate place to address this.
